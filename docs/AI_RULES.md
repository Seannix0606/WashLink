# AI Agent Mandatory Instructions

You are working on a production-grade enterprise application.

Before making ANY code changes, you MUST:

1. Read and follow ALL markdown files inside `/docs`
2. Treat `/docs` as the single source of truth
3. Never generate code that violates documented architecture
4. Prioritize:
   - readability
   - scalability
   - maintainability
   - SOLID principles
   - clean architecture
   - strict typing
   - modular design
5. Never use shortcuts, hacks, or temporary fixes
6. Never create duplicate logic
7. Always refactor duplicated code into reusable services/modules
8. Always explain:
   - what changed
   - why it changed
   - affected files
   - possible risks
9. Always preserve existing architecture patterns
10. If documentation conflicts exist, stop and ask for clarification

Mandatory workflow before coding:
- Read `/docs`
- Analyze affected architecture
- Plan implementation
- Implement
- Self-review against `/docs`
